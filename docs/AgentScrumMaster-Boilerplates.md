## 2. User Story Template

### Standard User Story

```
Title: [Brief descriptive title]

As a [type of user],
I want [goal/desire],
So that [benefit/value].

Acceptance Criteria:
- [ ] Given [context], when [action], then [expected result]
- [ ] Given [context], when [action], then [expected result]
- [ ] Given [context], when [action], then [expected result]

Technical Notes:
- [Implementation considerations]
- [API endpoints affected]
- [Database changes required]

Estimated Complexity: [1 | 2 | 3 | 5 | 8 | 13]
Dependencies: [List any blocking stories]
Labels: [feature | bugfix | tech-debt | spike]
```

```

Acceptance Criteria (Auto-Generated):
- [ ] [Derived from technical documentation via RAG]
- [ ] [Derived from similar past stories]
- [ ] [Derived from API contract / schema analysis]

Confidence Score: [0.0 - 1.0]
Source References:
  - [Document/page that informed this story]
  - [Past sprint story similarity: PROJ-XXX (87% match)]

Estimated Complexity: [Fibonacci value]
Estimation Basis: [Historical average for similar stories]
Risk Flags: [Any detected risks or ambiguities]
```

## 8. Integration Webhook Configuration

### GitHub Webhook Payload Handler

```yaml
github_events:
  pull_request:
    actions: [opened, closed, merged, review_requested]
    extract:
      - author
      - title
      - linked_issues
      - review_status
      - merge_conflicts

  push:
    branches: [main, develop]
    extract:
      - author
      - commit_messages
      - files_changed
      - ci_status

  issues:
    actions: [opened, closed, labeled]
    extract:
      - title
      - labels
      - assignees
      - milestone
```

### Jira Webhook Configuration

```yaml
jira_events:
  issue:
    events: [created, updated, deleted, moved]
    extract:
      - status_change
      - assignee_change
      - story_points
      - sprint_change
      - comment_added

  sprint:
    events: [started, completed, updated]
    extract:
      - sprint_goal
      - committed_issues
      - velocity
```

### Slack Event Subscription

```yaml
slack_events:
  message:
    channels: [standup, alerts, general]
    detect:
      - blocker_keywords: ["blocked", "stuck", "waiting on", "can't proceed"]
      - help_keywords: ["need help", "anyone available", "struggling with"]
      - completion_keywords: ["done", "merged", "deployed", "shipped"]
```
